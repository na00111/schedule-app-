# schedule-app-

1.3 Layer Architecture(Controller, Service, Repository)를 적절히 적용했는지 확인해 보고, 왜 이러한 구조가 필요한지 작성해 주세요.
 ->   한 클래스에 내용들을 다 구현하면 나주에 유지 보수하기 매우 어려워서 클래스를 나눠서 기능별로 분류하여 만듭니다.
 구조는 dto- > 컨트롤러 -> 서비스 -> 리포지토리 -> 서비스 -> 컨트롤러 -> dto
 dto에는 비밀번호 같은 보안이 필요한 내용은 적지 않고 정보를 보여줍니다
 컨트롤러에 정보들을 영구적으로 입력하여 보관합니다
 서비스는 비즈니스 로직을 처리합니다
 리포지토리는 데이터베이스를 저장합니다 리포지토리는 인터페이스로 설정합니다. 

2.@RequestParam, @PathVariable, @RequestBody가 각각 어떤 어노테이션인지, 어떤 특징을 갖고 있는지 작성해 주세요. ( 아직 정확하게 개념을 숙지하지 못했습니다)
->  @PathVariable 는 고유 식별자를 지정할 때 사용합니다. 생성 , 조회, 삭제 매핑 할 때 적어줍니다.
@RequestParam 는 검색 조건 ,필터링 , 비밀번호 검증에 사용됩니다
 @RequestBody 는 Dto를 받을 때 사용합니다.
